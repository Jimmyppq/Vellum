## ADDED Requirements

### Requirement: Repositories receive and return Pydantic models only
The system SHALL implement all repositories in `dal/app/repositories/`. Every repository method SHALL accept Pydantic models as input and return Pydantic models or Python primitives as output. SQLAlchemy `Row` objects SHALL NOT be exposed outside the repository class.

#### Scenario: Repository method returns Pydantic model
- **WHEN** `PromptsRepository.get_by_id(id)` finds a row
- **THEN** it SHALL return a `PromptResponse` Pydantic model, not a SQLAlchemy `Row`

#### Scenario: Repository returns None for missing records
- **WHEN** `PromptsRepository.get_by_id(id)` is called with a non-existent UUID
- **THEN** it SHALL return `None`

### Requirement: PromptsRepository implements five operations
The system SHALL implement `PromptsRepository` with: `create(data: PromptCreate) -> PromptResponse`, `get_by_id(id: UUID) -> PromptResponse | None`, `list(filters: PromptFilters) -> list[PromptResponse]`, `update_status(id: UUID, status: str) -> PromptResponse`, `delete(id: UUID) -> bool`.

#### Scenario: create generates UUID in Python
- **WHEN** `PromptsRepository.create` is called without providing an `id`
- **THEN** the inserted row SHALL have a UUID generated by `uuid.uuid4()` in Python

#### Scenario: update_status rejects invalid status values
- **WHEN** `PromptsRepository.update_status` is called with a status outside `{draft, approved, deprecated}`
- **THEN** it SHALL raise a `ValueError` before executing any query

### Requirement: PromptVersionsRepository implements five operations
The system SHALL implement `PromptVersionsRepository` with: `create(data: VersionCreate) -> VersionResponse`, `get_by_id(id: UUID) -> VersionResponse | None`, `list_by_prompt(prompt_id: UUID) -> list[VersionResponse]`, `get_active(prompt_id: UUID) -> VersionResponse | None`, `deactivate_all(prompt_id: UUID) -> None`.

#### Scenario: Activating a version deactivates previous ones atomically
- **WHEN** a new version is created with `is_active=True`
- **THEN** `deactivate_all` and the new insert SHALL execute within a single database transaction

#### Scenario: get_active returns None when no active version exists
- **WHEN** `get_active` is called for a prompt with no active version
- **THEN** it SHALL return `None`

### Requirement: ExecutionsRepository implements four operations
The system SHALL implement `ExecutionsRepository` with: `create(data: ExecutionCreate) -> ExecutionResponse`, `get_by_id(id: UUID) -> ExecutionResponse | None`, `update_status(id: UUID, status: str, output: dict | None) -> ExecutionResponse`, `list_by_prompt(prompt_id: UUID, limit: int, offset: int) -> list[ExecutionResponse]`.

#### Scenario: update_status sets completed_at when status is completed or failed
- **WHEN** `ExecutionsRepository.update_status` is called with `status=completed` or `status=failed`
- **THEN** the `completed_at` field SHALL be set to the current UTC timestamp

### Requirement: UsersRepository implements four operations
The system SHALL implement `UsersRepository` with: `create(data: UserCreate) -> UserResponse`, `get_by_id(id: UUID) -> UserResponse | None`, `get_by_email(email: str) -> UserResponse | None`, `assign_role(user_id: UUID, role_id: UUID) -> None`.

#### Scenario: get_by_email is case-insensitive
- **WHEN** `get_by_email` is called with an email in any case
- **THEN** it SHALL return the matching user regardless of case

#### Scenario: assign_role raises error for non-existent user or role
- **WHEN** `assign_role` references a user_id or role_id that does not exist
- **THEN** it SHALL raise a `ValueError` — no partial insert SHALL be committed

### Requirement: TranscriptsRepository and TranscriptVersionsRepository mirror Prompts structure
The system SHALL implement `TranscriptsRepository` with the same five operations as `PromptsRepository` (substituting `Transcript` models), and `TranscriptVersionsRepository` with the same five operations as `PromptVersionsRepository` (substituting `TranscriptVersion` models).

#### Scenario: Transcript status update rejects invalid values
- **WHEN** `TranscriptsRepository.update_status` is called with an unrecognized status
- **THEN** it SHALL raise a `ValueError`

### Requirement: ConnectorsRepository implements four operations
The system SHALL implement `ConnectorsRepository` with: `create(data: ConnectorCreate) -> ConnectorResponse`, `get_by_id(id: UUID) -> ConnectorResponse | None`, `list_active() -> list[ConnectorResponse]`, `set_active(id: UUID, is_active: bool) -> ConnectorResponse`.

#### Scenario: list_active returns only connectors with is_active=True
- **WHEN** `ConnectorsRepository.list_active()` is called
- **THEN** it SHALL return only connectors where `is_active` is `True`
